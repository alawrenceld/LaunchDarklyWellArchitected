# Pillar: Developer Experience & Velocity

> *Make the safe path the easy path.*

**Status: Phase 2 — draft scheduled, not yet written.**

This pillar covers SDK integration patterns, local dev with flags and AI Configs, unit and integration testing with flags and experiments, ephemeral environments, code review for LD changes, type-safety and codegen for flag and AI Config keys, IDE integrations, Terraform / IaC management of LD resources, the AI Config local iteration loop, and onboarding ramp.

## When this pillar will be most relevant

- You're integrating LaunchDarkly into a new codebase and want to do it well.
- Your team has friction around LD: testing, local dev, refactoring, finding flag references.
- You're standardizing LD practices across many teams or services.
